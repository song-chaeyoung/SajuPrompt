## Design Context

### Users
- 주요 사용자: 사주 해석 자체보다, 외부 생성형 AI(ChatGPT/Gemini/Claude)에 넣을 질문문을 정확하게 만들고 싶은 사용자.
- 사용 맥락: 짧은 시간 안에 개인 상황(생년월일/출생시간/관계/진로 고민)을 정리해 바로 복사-활용하려는 모바일 중심 사용.
- 핵심 과업(JTBD): 입력 실수를 줄이고, 목적에 맞는 질문문을 빠르게 생성해 다른 AI에 붙여넣어 해석 품질을 높인다.
- 근거: README 서비스 정의 및 사용 흐름, 주요 기능.
- 가정: 연령대/숙련도 분포는 문서에 명시가 없어, 초중급 사용자까지 포함하는 범용 UX를 우선한다.

### Brand Personality
- 성격 3단어: 현대적, 정제됨, 신뢰감.
- 전달 감정: 과장된 신비주의보다 차분한 확신과 명료한 안내.
- 참고 무드: 모던 미니멀 + 프리미엄.
- 근거: Linear CHA-95에 "모던 미니멀, 프리미엄 무드 (Co—Star / Pattern 앱 레퍼런스)" 명시.
- 가정(안티 레퍼런스): 과한 네온/글로우, 장식 중심 오컬트 톤, 정보 전달보다 분위기만 강조하는 스타일은 지양한다.

### Aesthetic Direction
- 테마: Light-first.
- 이유: CHA-95 팔레트가 아이보리 배경(`--color-bg: #FAF7F2`)과 딥 인디고 텍스트(`--color-text: #1A1A2E`) 중심으로 정의되어 있으며, 모바일 입력 폼의 가독성 우선 맥락과 맞다.
- 컬러 전략: 기존 `ivory / indigo / gold accent` 축은 유지하되, 실제 운용은 `warm ivory / ink indigo / muted brass accent`로 더 좁혀 해석한다.
- 강조색 사용 규칙: accent는 기본 CTA 전체 채움색이 아니라 ring, glow, 진행 상태, 소규모 강조 포인트에 제한적으로 사용한다.
- 선택 상태 규칙: 선택 카드와 보조 액션은 넓은 lavender fill보다 subtle tint, border, shadow로 상태를 구분한다.
- 토큰 중심 구현: 색상·타이포·간격·radius를 전역 토큰으로 통일하고, 개별 컴포넌트 커스텀 스타일은 최소화한다.
- 근거: Linear CHA-95 작업 범위(globals.css 토큰, tailwind 확장, 타이포/간격/radius 정의, 기존 컴포넌트 소급 적용)와 현재 `button`의 secondary 선택 상태 표현.

### Design Principles
1. 토큰 우선 일관성: 화면마다 새 스타일을 만들지 않고, 전역 토큰 체계 안에서만 변형한다.
2. 가독성 우선: 장식보다 입력 정확도와 결과문 이해 속도를 최우선으로 둔다.
3. 절제된 프리미엄: 강조색은 CTA/핵심 상태에만 사용해 정보 위계를 선명하게 만든다.
4. 모바일 퍼스트 흐름: 단계형 폼에서 현재 위치와 다음 행동이 즉시 보이도록 설계한다.
5. 결과 전달 최적화: "질문문 생성 → 복사 → 외부 AI 사용" 핸드오프를 가장 짧은 동선으로 유지한다.
- 근거: README 사용 흐름/AI 사용 방식, Linear CHA-95 및 관련 CHA-96(모바일 퍼스트 반응형).
